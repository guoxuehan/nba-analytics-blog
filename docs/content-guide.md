# コンテンツ作成ガイド

NBA COURT VISION の記事作成フローと命名規則をまとめたドキュメント。

---

## 記事作成の流れ

### 1. ネタ出し（`/today-topics`）

Claude Code で `/today-topics` を実行すると、今日のNBAニュース・トレンドに基づいた記事ネタを3件提案する。各ネタには以下が含まれる。

- タイトル案
- カテゴリ
- 旬である理由（なぜ今日書くべきか）
- 想定SEOキーワード
- データ収集の難易度（低/中/高）

### 2. 下書き生成（`/write-article`）

`/write-article` に記事テーマ・カテゴリ・キーワードを渡すと、以下の形式でMarkdown記事を生成する。

**出力フォーマット:**

```
---METADATA---
title: （記事タイトル）
slug: （英語ハイフン区切り）
category: （カテゴリ値）
tags: （カンマ区切り5〜7個）
excerpt: （100〜150字の要約）
---METADATA---

---BODY---
（Markdown本文 2,000〜3,000字）
---BODY---
```

**本文の構成:**
1. リード文（3〜4文）
2. データ分析セクション1（h2）
3. データ分析セクション2（h2）
4. 考察・深掘り（h2）
5. 筆者の視点（h2）
6. まとめ（h2）

### 3. 管理画面への入力

1. ブラウザで `/admin` にアクセスしてログイン
2. 「新規記事」をクリック
3. 生成された内容を以下のように分けて入力する

**METADATAセクションの入力先:**

| フィールド | 入力場所 |
|---|---|
| `title` | タイトル入力欄（最上部） |
| `slug` | Slug（URL）フィールド |
| `category` | カテゴリドロップダウン |
| `tags` | タグ入力欄（Enterで追加） |
| `excerpt` | 抜粋（リード文）テキストエリア |

**BODYセクションの入力先:**
- `---BODY---` と `---BODY---` の間のMarkdown本文をそのままMarkdownエディタに貼り付ける

### 4. サムネイル画像の設定

Unsplash（https://unsplash.com）から画像を取得して使用する。

**おすすめ検索ワード:** `basketball`, `NBA`, `sports arena`, `athlete`

画像URLを直接貼り付けるか、ダウンロードしてアップロードフォームからアップロードする。アップロードされた画像はSupabase Storageの `post-images` バケットに保存される。

### 5. 公開

- **「下書き保存」**: `published = false` で保存。フロントには表示されない。
- **「公開する」**: `published = true` に変更し、`published_at` を現在時刻にセット。即時フロントに反映される（ISRキャッシュが60秒以内に再生成）。

## カテゴリの定義

| カテゴリ値 | 表示名 | 内容 |
|---|---|---|
| `player_analysis` | 選手分析 | 個別選手のスタッツ・プレースタイル・成長分析 |
| `team_analysis` | チーム分析 | チーム全体の戦力・強み弱み・シーズン展望 |
| `tactics` | 戦術 | オフェンス/ディフェンスのシステム・戦術的トレンド |
| `data` | データ | 高度統計（PER, BPM, RAPM等）やデータ可視化を主体とした分析 |

---

## タグの命名規則

- **日本語を使用する**（例: `選手分析`, `プレーオフ`）
- **固有名詞はそのまま**（例: `SGA`, `ウェンバンヤマ`, `OKC`）
- **1記事あたり5〜7個**を目安にする
- **汎用的なタグを含める**（カテゴリに対応する語を1つ入れると良い）
- **短く具体的に**（`3ポイント` より `3P分析` など意味が明確な方が良い）

**良い例:** `MVP`, `SGA`, `ウェンバンヤマ`, `個人賞`, `プレーオフ`, `得点王`, `高度統計`

**避けるべき例:** `NBA`（広すぎる）, `分析`（単独では弱い）

---

## slugの命名規則

- **英語・数字・ハイフンのみ**を使用する
- **小文字**に統一する
- **スペースはハイフン**に変換する
- **記事内容を端的に表す**（SEOに有効）
- **一度公開したslugは変更しない**（URLが変わると被リンクが切れる）

**命名パターン:**
```
{トピック}-{補足}-{年度（任意）}

例:
mvp-race-2026-sga-vs-wembanyama-analysis
okc-bench-scoring-playoff-analysis
wembanyama-defensive-impact-2025-26
```

管理画面ではタイトルのASCII文字からslugを自動生成する。日本語タイトルの場合は `post-{タイムスタンプ}` にフォールバックするため、**手動で意味のあるslugに修正する**こと。

## SEOチェックリスト

記事公開前に以下を確認する。

### タイトル
- [ ] 30〜40文字程度
- [ ] 主要キーワードがタイトル前半に含まれている
- [ ] 「2026」などの年度を含める（時事性のある記事の場合）

### excerpt（抜粋）
- [ ] 100〜150文字
- [ ] キーワードを自然に含む
- [ ] 記事の価値（何がわかるか）が伝わる

### 本文
- [ ] h2見出しにキーワードが自然に含まれている
- [ ] 専門用語に補足説明がある
- [ ] 表・リストを活用して読みやすい構成になっている

### メタデータ
- [ ] slugが英語・ハイフン区切りで意味が通る
- [ ] タグが5〜7個設定されている
- [ ] カテゴリが適切に設定されている
- [ ] サムネイル画像が設定されている（OGP画像にも使用される）

## Claude Codeカスタムコマンド一覧

| コマンド | 主な用途 | 出力 |
|---|---|---|
| `/today-topics` | 記事ネタ出し | 優先順位付きネタ3件 |
| `/write-article` | 記事本文生成 | METADATA + BODYフォーマット |
| `/x-post` | SNS投稿文生成 | 3パターンのX（Twitter）投稿文 |
| `/weekly-report` | 週次振り返り | 掲載記事・SEO評価・改善案 |
| `/monthly-review` | 月次事業レビュー | KPI・カテゴリバランス・次月アクション |

コマンドの定義ファイルは `.claude/commands/` に格納されている。
